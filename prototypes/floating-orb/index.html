Floating Runtime UI

让主状态脱离菜单栏，变成一个独立悬浮的小图标

这一版把主交互改成 竞品 风格的 top floating orb。菜单栏只保留后台入口和设置； 真正的运行时反馈交给悬浮图标自己完成。按下时它扩张、呼吸，抬起时原地变成处理态，成功后收回。

Design Decision
菜单栏
不是主状态面板，只做入口。不要逼用户每次抬头去菜单栏确认现在在不在听。
悬浮层
独立存在，更像系统输入层。它可以始终低存在感悬在顶部，触发时才展开。
同位变形
按下、处理、成功都在同一个位置发生，不新增第二块 HUD，也不打断当前 app。
设置窗口
权限、键位、麦克风、日志等配置仍然收进独立设置窗口，不混进悬浮层。

1. UI 形态分层

状态切换

1
Idle

顶部保留一个很小的点。不是隐藏到菜单栏，而是维持“始终可见但不打扰”的存在感。

2
Press Down

点扩张成小球，出现明显呼吸感，提示已经在听。这里不出现设置或复杂文字。

3
Release

同位变形成处理态，位置不变。用户知道系统确实接到了结束信号。

4
Success / Fallback / Error

最后只做一个很短的结果确认，然后收回顶部待机点。失败必须给补救语义。

这版已经改成更接近 竞品 的位置：底部偏上一点，默认更小，待机态几乎只是一个点。 现在重点只剩两件事：待机是不是还要更小，处理态要不要再收一点。

3. 状态设计

v1 不需要很多状态，但必须让用户一眼看懂“正在听 / 正在处理 / 成功 / 失败”。下方是建议的状态集合。

Idle Orb

始终可见 低存在感

一个很小的待机点。让用户知道工具在线，但不做常驻浮条。

Listening Orb

按下触发键 立即跳动

点扩成球并开始呼吸。建议不用复杂声波，靠尺寸和节奏变化就够了。

Processing Capsule

抬起后 原地变形

视觉从球变成胶囊，但位置不变。表示系统已停止录音并在处理中。

Success

插入成功 短暂确认

变绿或显示 Done，约半秒后收回。不要再弹结果窗口。

Fallback

AX 失败 要能补救

例如“已复制，可手动粘贴”。这比纯报错更实用，也更像可用产品。

Error

网络或权限 短错误

只给一句短错误，不展开成长面板。更详细的东西留给设置页或诊断页。
