# 音键 V1 收口清单

最后更新：2026-04-15

## 1. 目标

`音键 v1` 只做一件事：

`按住 -> 说话 -> 松开 -> 很快得到干净文本 -> 稳定写回当前输入框`

这不是一个“智能输入平台”版本。

这不是一个“选中文本改写 + 问答 + 翻译”版本。

这不是一个“功能尽量多”的版本。

这是一版只围绕 `语音转文本` 主链路做收口的产品版本。

## 2. 当前判断

目前项目已经具备基础闭环：

- 菜单栏常驻
- 全局触发
- 录音
- ASR
- cleanup
- 文本写回
- 最小 fallback

但当前仍然存在两个明显问题：

1. 产品范围没有完全收口。
2. 核心链路周围还有若干半成品和实验项没有处理干净。

因此当前阶段的正确方向不是继续加功能，而是：

`删掉非 v1 心智，补齐核心链路缺口，把唯一黄金路径做到稳定、顺滑、可日用。`

## 3. V1 产品边界

### 3.1 V1 必须交付

- 菜单栏常驻
- 一个稳定可用的全局触发键
- hold-to-talk
- 录音成功率稳定
- 语音转文本
- 文本基础整理
- 稳定写回当前输入框
- 写入失败时用户可恢复
- 权限缺失时有明确提示

### 3.2 V1 明确不做

- selected-text edit
- ask mode
- translation mode
- 个人词典
- app profile UI
- 历史面板
- 多设备同步
- 团队能力
- 复杂工作流编排

这些能力不是永远不做，而是这版不应该进入主开发目标。

## 4. 收口原则

### 4.1 一个默认主链路

用户不需要理解：

- offline / realtime
- cleanup prompt profile
- 模型选择
- 技术 fallback 策略

用户只应该感知到：

`按住说话，松开出字。`

### 4.2 稳定优先于可玩性

这一版优先级：

1. 稳定录音
2. 稳定出字
3. 稳定写回
4. 失败可恢复
5. 再谈速度和细节体验

### 4.3 半成品要么做完，要么先藏起来

凡是用户能看到但还没做完整的能力，都应视为产品负债。

## 5. P0 必须完成

### P0-1 统一文档口径

要做：

- 把长期 PRD 和当前 v1 范围对齐
- 明确写清：当前版本只做语音转文本主链路
- 把 selected-text、词典、翻译、历史面板等从 v1 必做中移出

完成标准：

- 不再出现“PRD 说必做，但当前实现明确不做”的冲突

### P0-2 固定唯一默认链路

要做：

- 第一版只保留一个默认识别链路
- realtime 作为内部实验能力，不作为用户理解负担
- cleanup 默认开启，不让用户理解太多中间概念

完成标准：

- 普通用户不需要做技术选择
- 设置页不再暴露明显的实验性心智

### P0-3 固定唯一默认触发键

要做：

- 第一版明确一个最稳定的默认触发键
- 不再同时承载“未来想做的键”和“当前实际最好用的键”

完成标准：

- 新用户第一次使用时，不需要思考“该选哪个键”

### P0-4 失败恢复做成用户能力

要做：

- 至少支持“最近一次结果重新插入”
- 至少支持“最近一次结果复制到剪贴板”
- 写回失败时，用户知道结果还在，不会丢

完成标准：

- 出现写入失败时，用户仍能在 1 到 2 步内拿回文本

### P0-5 边界行为补齐

要做：

- 当前没有可写输入框时，给明确提示
- 当前控件只读时，给明确提示
- 权限缺失时，提示清楚是麦克风问题还是辅助功能问题
- 写回失败时，给恢复动作，而不是仅给错误

完成标准：

- 常见失败不再表现为“没反应”或“用户不知道发生了什么”

### P0-6 设置页收口

要做：

- 去掉明显开发者导向的配置暴露
- 去掉实验性质的模型配置入口
- 保留真正影响主体验的少数设置

建议保留：

- 触发键
- 麦克风权限状态
- 辅助功能权限状态
- 最小的写回回退开关

完成标准：

- 设置页是产品设置，不是调试面板

## 6. P1 应该完成

### P1-1 麦克风设备能力收口

要做二选一：

- 要么真正做完设备选择
- 要么在 v1 里移除该设置，只用系统默认输入

完成标准：

- 用户看见的设置项都是真实生效的

### P1-2 敏感输入保护

要做：

- 密码框不自动写入
- 敏感输入场景至少有防误触策略

完成标准：

- 用户不会在明显不该写入的位置误触发完整链路

### P1-3 最小结果留存入口

要做：

- 菜单栏里提供“最近一次结果”入口，或等价入口
- 不要求做完整历史面板

完成标准：

- 用户能找回刚刚那次结果

### P1-4 主观流畅度打磨

要做：

- 权限引导更短
- HUD 信息更克制
- 失败提示更可执行
- 松键后的等待感更低

完成标准：

- 用户主观感受接近“一个顺手的输入工具”，而不是“在跑一条 AI 流程”

## 7. P2 可以晚一点

这些不是当前版本阻塞项：

- realtime partial 文本显示
- 选中文本 edit / ask / translate
- 个人词典
- app-specific tone/profile
- 历史面板
- 翻译模式

它们都属于后续增强项，不应在 v1 收口阶段抢优先级。

## 8. 建议保留的用户心智

第一版应该只让用户形成下面这套认知：

- 这是一个菜单栏语音输入工具
- 按住说话，松开出字
- 出来的是整理过的可直接发送文本
- 大多数时候能直接写进当前输入框
- 偶尔失败时，结果也不会丢

只要这 5 条成立，v1 就成立。

## 9. 一句话版本

`音键 v1 的任务不是做强，而是做顺。`

只把 `语音转文本` 这一条主链路做到稳定、干净、可恢复，就已经足够成为第一版产品。
