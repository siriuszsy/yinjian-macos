# tinyTypeless 产品需求文档

最后更新：2026-04-10

说明：

这份文档保留的是产品长期方向。

当前实际实现范围请以 [V1 系统设计](./v1-system-design.md) 和 [技术架构](./technical-architecture.md) 为准。

## 1. 产品定义

tinyTypeless 是一个 `macOS 系统级智能语音输入工具`。

它的工作不是保留原始口语，而是把用户说出来的话直接变成“可以发出去、可以保存、可以继续编辑”的干净文本。

## 2. 目标

替代当前本地的 Typeless。

替代的标准不是“有相同按钮”，而是：

- 日常写消息、邮件、文档时愿意长期打开 tinyTypeless
- 在高频场景下主观体验不输 Typeless
- 当用户想改写、翻译、总结选中文本时，不需要切到别的工具

## 3. 目标用户

第一阶段只有一个目标用户：

`你自己`

这反而是好事。目标非常清楚，不需要先猜大众市场。

高频场景：

- 聊天输入
- 写邮件
- 写文档
- 改写一段已经存在的文字
- 看网页或文档时快速总结、解释、翻译

## 4. Jobs To Be Done

### 4.1 即时听写

当我不想打字时，我希望按住一个热键，说完就把文字落到当前输入框里，而且结果足够干净，像我自己认真打出来的一样。

### 4.2 语音改写

当我选中一段文字时，我希望直接说“更简洁一点”“翻成英文”“改得更礼貌”，系统马上替我改掉，而不是弹出一个聊天窗口。

### 4.3 只读文本问答

当我在网页或文档里看到一段内容时，我希望选中它然后提问，比如“总结一下”“这是什么意思”，答案要来得快，不打断当前流。

## 5. 产品原则

### 5.1 输入层优先

这是输入层，不是聊天产品。

默认动作应该是：

- 写进当前输入框
- 替换当前选中文本
- 在当前上下文里完成动作

而不是跳到独立主窗口。

### 5.2 默认生成干净文本

输出默认应该比逐字稿更接近“最终稿”。

最少要自动处理：

- 口头禅
- 重复
- 中途改口
- spoken list
- 语气整理

### 5.3 快比全重要

用户天天用的工具，最怕摩擦。

必须优先优化：

- 启动速度
- partial 反馈
- 结束稳定性
- 文本注入成功率

### 5.4 失败要可恢复

任何跨应用工具都会遇到异常。

因此必须有：

- 插入失败回退
- 权限缺失提示
- 最近结果可重发
- 明确的日志与调试信息

## 6. V1 范围

### 6.1 必做

- macOS 菜单栏常驻
- 全局热键
- hold-to-talk 模式
- 当前 focused input 的文本插入
- 选中文本读取与替换
- 流式 partial
- final clean-up
- 自动去口头禅、去重复、改口收敛
- spoken list 自动格式化
- 个人词典
- 每个 app 的语气配置
- 历史记录本地保存
- 选中文本的 edit / ask / translate
- translation mode

### 6.2 可以晚一点

- 多设备同步
- 团队功能
- iOS / Windows 客户端
- Web 控制台
- 超长录音文件转写

## 7. 非目标

V1 明确不做：

- 输入法
- 唤醒词助手
- 独立笔记产品
- 会议记录产品
- 面向公众的订阅和计费系统

## 8. 关键体验指标

### 8.1 延迟

- 热键按下到录音开始：`<100ms`
- 第一段 partial 出现：`<400ms`
- 用户松键到最终文本插入：`<700ms`

### 8.2 成功率

- 常用 app 集合插入成功率：`>95%`
- selected-text 替换成功率：`>95%`

### 8.3 文本质量

主观上至少达到：

- 不明显比 Typeless 更啰嗦
- 不明显比 Typeless 更机械
- 不明显比 Typeless 更像逐字稿

## 9. 关键用户流

### 9.1 全局听写

1. 用户在任意输入框里聚焦光标
2. 按住热键开始说话
3. 系统立刻显示 listening 状态
4. partial 很快出现
5. 用户松开热键或停顿结束
6. 系统完成 clean-up
7. 最终文本插入当前光标位置

### 9.2 选中文本改写

1. 用户在任意可编辑文本里选中一段文字
2. 按住热键并说“更简洁一点”
3. 系统读取选中文本作为上下文
4. 生成新文本
5. 直接替换原文

### 9.3 选中文本问答

1. 用户选中网页或 PDF 中的一段文本
2. 按住热键说“总结一下”
3. 系统读取所选内容
4. 结果以浮层或插入方式返回

## 10. 关键边界情况

- 没有 focused text field
- 当前 app 不支持 Accessibility 读写
- 当前字段是密码框或敏感输入框
- 蓝牙耳机切换中
- 用户说到一半改口
- 中英混说
- 专有名词拼错
- 权限被撤销
- selected text 是只读内容

## 11. 成功定义

当下面这件事发生时，tinyTypeless 才算真的成了：

`你已经把 Typeless 关掉，并且连续几天都不想再开回去。`
